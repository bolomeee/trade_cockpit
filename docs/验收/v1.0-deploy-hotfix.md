**日期**：2026-04-20

**部署方式**：`docker compose build` + `docker compose up -d`

---

## 问题 1：容器添加 watchlist 失败

**症状**：localhost:8080 添加 watchlist 时 backend 500，日志显示 `RuntimeError: FMP_API_KEY not set`

**根因**：`docker-compose.yml` backend service `environment` 段只透传 `POLYGON_API_KEY`，漏了 `FMP_API_KEY`；D034 迁移数据源到 FMP 后没更新部署配置

**修复**：
- `docker-compose.yml` 补 `- FMP_API_KEY=${FMP_API_KEY}`
- `docker compose up -d` 触发容器 recreate 加载新 env

**验收**：watchlist 搜索 + 添加成功，signals 能正常返回

---

## 问题 2：MarketOverviewBar 缺 NASDAQ 100

**症状**：`/api/market/overview` 返回只有 SPX + TNX，NDX 缺失；`/api/logs?level=ERROR` 显示 `NDX refresh failed: 402 Payment Required`（请求路径 `...symbol=^NDX...`）

**根因**：FMP Starter 套餐不包含 `^NDX`（Nasdaq 授权指数，通常在 Premium 及以上）

**方案分析**：
- ❌ 升级 Premium — 为单个指标升档不划算
- ❌ `^IXIC`（NASDAQ Composite）替代 — 用户明确否决："纳指看宽泛了没用"
- ✅ **QQQM ETF 价格替代** — Starter 覆盖，tracker 与 NDX 日内方向 >99% 同步

**修复（见 D037）**：
- `backend/app/services/market_refresh_service.py`：`_DB_TO_FMP_INDEX["NDX"]` 从 `"^NDX"` → `"QQQM"` + 备注说明
- `backend/tests/test_market_refresh.py`：3 处 `"^NDX"` → `"QQQM"`，保留 `called_symbols` 断言
- DB symbol / 前端显示名 / API 合约 **均不变**

**验收**：
- `pytest tests/test_market_refresh.py` 8/8 通过
- backend 镜像 rebuild → `docker compose up -d backend` → `POST /api/data/refresh` → `GET /api/market/overview` 三行齐全：
  - SPX 7126.05 (+1.20%)
  - NDX 267.13 (+1.30%)（QQQM 价格）
  - TNX 4.26% (-1.39%)
- 浏览器 localhost:8080 顶栏三指标正常显示

## 已知说明

- NDX 行的 `close` 字段是 QQQM ETF 价格（~$267），**不是**指数点数（~18000）；`changePct` 仍然忠实反映当日 NASDAQ 100 方向，前端信息价值等同
- 若 FMP 将来对 `^NDX` 放开 Starter 访问，一行配置即可回切 `^NDX`
