## D：APScheduler EXPIRED（F206-b2）

### D-1：验证 expirer 函数（直接调用）

> 最简验证方式：在 Python shell 中直接调用。

```bash
cd backend && uv run python3 -c "
from datetime import date
from app.database import SessionLocal
from app.services.cockpit.pending_order_expirer import expire_due_pending_orders
from app.repositories.pending_order_repository import PendingOrderRepository

# 先创建一个过期的条件单（通过 repo 直接写 expiration_date=昨天）
db = SessionLocal()
repo = PendingOrderRepository(db)

# 创建一个已过期的条件单
row = repo.create({
    'ticker': 'TEST',
    'setup_type': 'BREAKOUT',
    'entry_price': 100.0,
    'stop_price': 90.0,
    'shares': 10,
    'expiration_date': date(2026, 4, 1),  # 很久以前
})
db.commit()
print(f'Created ACTIVE order id={row.id}, expiration_date=2026-04-01')

# 调用 expirer
n = expire_due_pending_orders(db, today=date.today())
print(f'Expired: {n} rows')

# 验证状态变了
updated = repo.get_by_id(row.id)
print(f'Order {row.id} status is now: {updated.status}')
db.close()
"
```

| 期望 | 实际 | 状态 |
|------|------|------|
| `Expired: 1 rows` | | ⬜ |
| order status 变为 `EXPIRED` | | ⬜ |
